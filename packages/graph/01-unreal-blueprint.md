# Unreal Blueprint 深度解析

> 工业级可视化脚本系统 | 图灵完备 | 字节码编译 | K2Node 扩展机制

---

## 1. 概述与定位

Unreal Blueprint（蓝图）是 Epic Games 在 2014 年随 UE4 推出的可视化脚本系统。其设计目标是：

1. **让设计师能编程**：不写 C++ 代码即可实现游戏逻辑
2. **与 C++ 无缝互通**：`UFUNCTION(BlueprintCallable)` 即可暴露原生函数
3. **真正编译执行**：不是解释执行，而是编译为字节码在专用 VM 上运行
4. **图灵完备**：变量、函数、宏、循环、继承等全部支持

**重要区分**：Blueprint ≠ Kismet（UE3 的可视化脚本，已废弃）。Blueprint 是全新设计的系统，复用了 Tim Sweeney 在 1998 年为 UnrealScript 编写的 VM。

## 2. 核心架构

### 2.1 五大类层次

```
UBlueprint (UObject)
  │
  ├── UEdGraph          // 可视化编辑器中的图
  │     ├── UEdGraphNode[]           // 节点基类
  │     │     └── UK2Node            // Blueprint 专用节点基类
  │     │           ├── UK2Node_CallFunction    // 调用 C++ 函数
  │     │           ├── UK2Node_IfThenElse     // 条件分支
  │     │           ├── UK2Node_ForEach        // 循环
  │     │           ├── UK2Node_SpawnActor     // 生成 Actor
  │     │           └── ...                    // 100+ 内置类型
  │     └── UEdGraphPin[]           // 输入/输出引脚
  │
  └── UBlueprintGeneratedClass  // 编译后的真正 UClass
        └── UFunction
              └── Script: TArray<uint8>  // 字节码数组
```

### 2.2 Pin（引脚）系统

Pin 是 Blueprint 最核心的概念，分为两类：

| 类型 | 颜色 | 含义 |
|------|------|------|
| **Exec Pin** | 白色 | 控制流，执行顺序 |
| **Data Pin** | 彩色 | 数据，按类型着色 |

Data Pin 类型与颜色对应：

| 类型 | 颜色 | 示例 |
|------|------|------|
| Boolean | 绿色 | `true` / `false` |
| Integer | 红色 | `42` |
| Float | 橙色 | `3.14` |
| String | 黄色 | `"Hello"` |
| Object | 蓝色 | `APawn*` |
| Class | 紫色 | `ACharacter::StaticClass()` |

### 3.3 Pure vs Impure 节点

这是 Blueprint 最重要的设计之一：

```cpp
// Pure 节点 - 无 exec pin，无副作用，按需求值（类似数据流）
// 特征：UK2Node::IsPure() == true
// 用途：数学运算、类型转换、Getter

// Impure 节点 - 有 exec pin，有副作用，按执行顺序运行（类似控制流）
// 特征：UK2Node::IsPure() == false
// 用途：Spawn Actor、Print String、Set Variable
```

## 4. 编译管线深度解析

这是 Blueprint **最精华也最复杂的部分**。它不是"图解释器"，而是"图编译器"。

### 4.1 编译流程图

### 4.2 字节码指令系统

字节码是单字节枚举（因此叫 byte code），常用指令：

```cpp
// 简化自 Engine/Source/Runtime/Engine/Classes/Kismet/
namespace EExprToken {
    // 基础运算
    const uint8 EX_LocalVariable       = 0x01;  // 读取局部变量
    const uint8 EX_InstanceVariable     = 0x02;  // 读取成员变量
    const uint8 EX_Assignment           = 0x03;  // 变量赋值
    const uint8 EX_Add_Int             = 0x04;  // 整数加法
    const uint8 EX_Subtract_Int        = 0x05;  // 整数减法
    const uint8 EX_CallMathFunction    = 0x06;  // 调用数学函数

    // 控制流
    const uint8 EX_Jump                = 0x07;  // 无条件跳转
    const uint8 EX_JumpIfNot           = 0x08;  // 条件跳转
    const uint8 EX_Return              = 0x09;  // 函数返回

    // 函数调用
    const uint8 EX_CallFunction        = 0x0A;  // 调用普通函数
    const uint8 EX_CallDelegate        = 0x0B;  // 调用委托

    // 对象操作
    const uint8 EX_SpawnActor          = 0x0C;  // 生成 Actor
    const uint8 EX_DestroyActor        = 0x0D;  // 销毁 Actor
    const uint8 EX_Array_GetByRef      = 0x0E;  // 数组按引用获取

    // 调试
    const uint8 EX_DebugBreak          = 0xFF;  // 断点
}
```

### 4.3 ExpandNode 示例：IfThenElse 展开

```cpp
// K2Node_IfThenElse 的 ExpandNode 实现逻辑
void UK2Node_IfThenElse::ExpandNode(
    FKismetCompilerContext& CompilerContext,
    UEdGraph* SourceGraph
) {
    // 1. 创建 Branch 节点（底层节点）
    UK2Node_Branch* BranchNode = CompilerContext.SpawnIntermediateNode<
        UK2Node_Branch>(this, SourceGraph);

    // 2. 把本节点的 Condition Pin 连接到 Branch 的 Condition
    CompilerContext.MovePinLinksToIntermediate(
        *FindPin(TEXT("Condition")),
        *BranchNode->FindPin(TEXT("Branch"))
    );

    // 3. 本节点的 True 分支连接到 Branch 的 True
    CompilerContext.MovePinLinksToIntermediate(
        *FindPin(TEXT("Then")),
        *BranchNode->FindPin(TEXT("True"))
    );

    // 4. 本节点的 False 分支连接到 Branch 的 False
    CompilerContext.MovePinLinksToIntermediate(
        *FindPin(TEXT("Else")),
        *BranchNode->FindPin(TEXT("False"))
    );

    // 5. 把 exec pin 连接到 Branch 的 exec
    UEdGraphPin* ExecPin = GetExecPin();
    UEdGraphPin* BranchExec = BranchNode->GetExecPin();
    ExecPin->MoveToIntermediate(BranchExec, ...);

    // 6. 断开本节点所有链接（已被展开）
    BreakAllNodeLinks();
}
```

## 5. 自定义 K2Node 实战

### 5.1 完整自定义节点：IsValid 判断

```cpp
// MyK2Node_IsValid.h
UCLASS()
class UMyK2Node_IsValid : public UK2Node {
    GENERATED_BODY()

public:
    // 节点标题
    virtual FText GetNodeTitle(ENodeTitleType::Type TitleType) const override {
        return LOCTEXT("IsValid", "Is Valid");
    }

    // 工具提示
    virtual FText GetTooltipText() const override {
        return LOCTEXT("Tooltip", "Checks if an object reference is valid");
    }

    // 创建默认引脚
    virtual void AllocateDefaultPins() override {
        Super::AllocateDefaultPins();

        // Exec 输入
        UEdGraphPin* ExecIn = CreatePin(
            EGPD_Input,
            UEdGraphSchema_K2::PC_Exec,
            FName(TEXT("Execute"))
        );

        // 对象输入
        UEdGraphPin* ObjectIn = CreatePin(
            EGPD_Input,
            UEdGraphSchema_K2::PC_Object,
            UObject::StaticClass(),
            FName(TEXT("Object"))
        );

        // Exec 输出 - Valid 分支
        UEdGraphPin* ValidOut = CreatePin(
            EGPD_Output,
            UEdGraphSchema_K2::PC_Exec,
            FName(TEXT("Is Valid"))
        );

        // Exec 输出 - Invalid 分支
        UEdGraphPin* InvalidOut = CreatePin(
            EGPD_Output,
            UEdGraphSchema_K2::PC_Exec,
            FName(TEXT("Is Not Valid"))
        );
    }

    // 核心！展开为本节点组合
    virtual void ExpandNode(
        FKismetCompilerContext& CompilerContext,
        UEdGraph* SourceGraph
    ) override {
        Super::ExpandNode(CompilerContext, SourceGraph);

        // 获取引脚
        UEdGraphPin* ObjectPin = FindPin(TEXT("Object"));
        UEdGraphPin* ValidOut = FindPin(TEXT("Is Valid"));
        UEdGraphPin* InvalidOut = FindPin(TEXT("Is Not Valid"));

        // 1. 创建 Branch 节点
        UK2Node_CallFunction* BranchNode =
            CompilerContext.SpawnIntermediateNode<UK2Node_CallFunction>(
                this, SourceGraph);

        // 2. 设置为 UKismetSystemLibrary::IsValid
        BranchNode->FunctionReference.SetExternalMember(
            GET_FUNCTION_NAME_CHECKED(UKismetSystemLibrary, IsValid),
            UKismetSystemLibrary::StaticClass()
        );
        BranchNode->AllocateDefaultPins();

        // 3. 迁移 Object 引脚
        CompilerContext.MovePinLinksToIntermediate(
            *ObjectPin,
            *BranchNode->FindPin(TEXT("Object"))
        );

        // 4. 迁移 Valid 分支
        CompilerContext.MovePinLinksToIntermediate(
            *ValidOut,
            *BranchNode->FindPin(TEXT("True"))
        );

        // 5. 迁移 Invalid 分支
        CompilerContext.MovePinLinksToIntermediate(
            *InvalidOut,
            *BranchNode->FindPin(TEXT("False"))
        );

        // 6. 断开本节点
        BreakAllNodeLinks();
    }

    // 节点类别（编辑器中显示在哪）
    virtual FName GetNodeCategory() const override {
        return FName(TEXT("Flow Control"));
    }
};
```

## 6. 执行模型

### 6.1 Blueprint VM 执行循环

编译后的字节码在 UE 的 Script 虚拟机上执行：

```cpp
// 简化自 Engine/Source/Runtime/Engine/Private/KismetVM/
void FKismetVM::ExecuteUbergraph(UObject* Object, int32 StartNode) {
    // 1. 获取编译后的 UFunction
    UFunction* Function = Object->FindFunctionChecked(FName("ExecuteUbergraph"));

    // 2. 创建 Stack + Context
    FFrame Stack(Function, Object, NULL);
    KismetExecutionContext Context(&Stack);

    // 3. 字节码解释循环
    while (true) {
        uint8 Opcode = *Stack.Code++;

        switch (Opcode) {
            case EX_Jump: {
                int32 Offset = ReadInt(&Stack);
                Stack.Code = Function->Script.GetData() + Offset;
                break;
            }
            case EX_JumpIfNot: {
                int32 Offset = ReadInt(&Stack);
                bool Condition = ReadBool(&Stack);
                if (!Condition) {
                    Stack.Code = Function->Script.GetData() + Offset;
                }
                break;
            }
            case EX_CallFunction: {
                // 读取函数引用，调用 C++ 实现
                FName FuncName = ReadName(&Stack);
                UFunction* TargetFunc = FindFunction(FuncName);
                Context.ProcessInternalCall(TargetFunc);
                break;
            }
            case EX_Return: {
                return;  // 函数结束
            }
            // ... 其他指令
        }
    }
}
```

### 6.2 循环在字节码中的实现

Blueprint 支持 For、While、ForEach 等循环。循环不是原生支持的，而是通过**跳转指令**实现：

```
// Blueprint 源码 (简化):
// for (int i = 0; i < 10; i++) { Print(i); }

// 编译后字节码:
0: EX_LocalVariable  i        // 声明 i
1: EX_Assignment    i = 0    // i = 0
2: EX_JumpIfNot     8        // 如果 i >= 10，跳到 8（循环结束）
3: Call   Print(i)           // 调用 Print(i)
4: EX_Add_Int       i, 1     // i = i + 1
5: EX_Assignment    i         // 写回 i
6: EX_Jump          2         // 跳回 2
7: (循环体结束)
8: (后续代码)
```

## 9. 优缺点分析

### 9.1 优点

1. **图灵完备 + 高性能**：编译为字节码，比解释执行快 10-100 倍
2. **与 C++ 完全互通**：任何 `UFUNCTION` / `UPROPERTY` 都可暴露
3. **强类型系统**：Pin 类型不匹配时编辑器直接拒绝连接
4. **完整的 OOP**：类、继承、接口、成员变量、构造逻辑
5. **工业级稳定性**：Epic 十年打磨，百万行代码游戏验证

### 9.2 缺点

1. **VM 老旧**：继承自 1998 年 UnrealScript，性能上限低于原生 C++
2. **图复杂度爆炸**：>100 节点的图几乎无法维护
3. **强耦合 UE**：无法独立使用
4. **编译时间**：大型蓝图编译可能需要数十秒
5. **调试困难**：节点级调试不够精细

## 10. 核心设计模式

### 10.1 模式一：ExpandNode 展开模式

所有复杂节点在编译时"展开"为简单节点组合。这是 Blueprint 的核心编译策略：

```
K2Node (编译期)
    ↓ ExpandNode()
简单节点组合 (运行时)
```

### 10.2 模式二：Functor 处理函数

每种节点类型配套一个 `FNodeHandlingFunctor`，负责把节点翻译为 IR：

```cpp
class FKCHandle_VariableSet : public FNodeHandlingFunctor {
    // 1. 注册本类型处理的节点
    virtual void RegisterNets(FKismetFunctionContext& Context,
                              UEdGraphNode* Node) override;

    // 2. 生成中间表示
    virtual void GenerateKOBSC(FKismetFunctionContext& Context,
                              UEdGraphNode* Node,
                              TArray<FBKOperator>& Ops) override;

    // 3. 编译为字节码
    virtual void Compile(FKismetFunctionContext& Context,
                         const FBKOperator& Op) override;
};
```

### 10.3 模式三：MemberReference 跨模块调用

```cpp
// 引用其他 Blueprint 的函数/变量
FMemberReference MemberReference;

// 编译时解析：
// 1. 查找目标 UBlueprintGeneratedClass
// 2. 查找目标 UFunction/UProperty
// 3. 生成跨类调用的字节码
```

## 13. 小结

Unreal Blueprint 是**工业级可视化脚本的标杆**，其设计精髓：

1. **ExpandNode 展开模式**：复杂节点编译时展开为简单组合
2. **双 Pin 系统**：exec（控制流）+ data（数据流）分离
3. **编译为字节码**：高性能执行，而非解释执行
4. **Pure vs Impure**：数据流与控制流的哲学分离
5. **与 C++ 无缝互通**：任何 `UFUNCTION` 即可暴露

理解 Blueprint 的编译管线是学习"如何设计可视化编程语言"的最佳实践。
